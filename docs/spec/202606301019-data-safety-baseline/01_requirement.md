# 数据安全最小闭环需求规格

## 背景

XZY Arbor 已完成轻量导图 MVP，具备节点编辑、拖拽整理、备注、JSON 打开保存等基础闭环。下一阶段的首要目标不是继续堆叠新功能，而是降低真实使用时的丢数据、误操作和坏文件风险。

本规格对应路线图中的 **P0-A 数据安全最小闭环**，聚焦用户每天都会遇到的高风险路径：误删误拖后可恢复、切换文件前能明确选择保存或放弃、打开坏文件不会覆盖当前内容、窗口能清楚显示当前文件和未保存状态。

## 目标

- 支持撤销 / 重做最近的导图内容和结构变更。
- 用鼠标友好的按钮式确认替代输入 `save` / `discard` / `cancel` 的提示框。
- 打开无效或不兼容文件时，当前导图保持不变，并给出明确中文提示。
- 在系统窗口标题和应用内标题中显示当前文件名与未保存状态。
- 保持当前 JSON 文件格式兼容，不因为本轮功能升级文件版本。

## 范围内

### 撤销 / 重做

- 支持撤销最近的节点内容和结构变更。
- 支持重做刚刚撤销的变更。
- 第一版历史上限为 50 步。
- 以下操作进入历史：
  - 新增同级节点。
  - 新增子节点。
  - 删除节点及其子树。
  - 拖拽移动节点。
  - 修改节点文本。
  - 修改节点备注。
  - 折叠或展开节点。
- 撤销 / 重做后应恢复对应的导图文档内容，并尽量保持用户正在关注的节点处于合理选中状态。
- 用户执行新的编辑操作后，应清空原有重做栈。
- 工具栏提供可点击的撤销 / 重做按钮；快捷键 `Ctrl+Z`、`Ctrl+Y` 和 `Ctrl+Shift+Z` 作为辅助能力。

### 保存确认流程优化

- 在新建、打开、关闭应用前，如果当前导图有未保存更改，应显示明确的按钮式确认。
- 确认选项为：
  - 保存。
  - 放弃更改。
  - 取消。
- 用户选择保存时，如果保存成功，继续原操作。
- 用户选择保存时，如果取消保存路径选择或保存失败，不继续执行会丢失当前内容的操作。
- 用户选择放弃更改时，继续原操作并丢弃当前未保存内容。
- 用户选择取消时，保留当前导图并中止原操作。
- 该确认流程应适用于新建、打开和关闭应用三个入口。

### 文件错误与兼容提示增强

- 打开文件失败时，当前内存导图、文件路径和未保存状态保持不变。
- 对以下情况给出清晰中文提示：
  - 所选文件不是合法 JSON。
  - 文档版本不受支持。
  - 文档结构非法。
  - 文件读取失败。
- 错误提示应说明当前导图没有被替换。
- 保存失败时，应保留当前导图和未保存状态，并提示用户保存未完成。

### 窗口标题与文件状态

- 系统窗口标题显示当前文件名。
- 未保存时，系统窗口标题显示脏状态标记。
- 应用工具栏中的当前文件显示与系统窗口标题保持一致。
- 未保存新文件显示为未命名导图。
- 打开或另存为后，标题立即切换为对应文件名。

## 范围外

- 不实现自动保存、草稿恢复或异常崩溃恢复。
- 不实现最近文件列表。
- 不改变导图 JSON 文档结构，不新增 `version: 2`。
- 不把画布平移、缩放纳入撤销 / 重做历史。
- 不实现多文件、多标签或跨窗口编辑。
- 不实现系统剪贴板、复制粘贴节点或搜索定位。
- 不引入命令面板；本轮以可见按钮和直接交互为主。
- 不引入 SQL、数据库、云同步或后端服务。

## 验收标准

- 用户误删非根节点后，可以点击撤销按钮恢复该节点及其子树。
- 用户误拖节点后，可以撤销恢复到拖拽前的父子关系和同级顺序。
- 用户撤销后执行新的编辑操作，原重做记录不再可用。
- 用户有未保存更改时点击新建、打开或关闭应用，会看到保存 / 放弃更改 / 取消三个明确按钮。
- 用户在确认流程中选择保存但保存失败时，当前导图不会被新建、打开或关闭流程替换。
- 用户打开坏文件时，当前导图保持不变，并看到说明原因和“当前导图未被替换”的提示。
- 用户打开或保存文件后，系统窗口标题和应用内标题显示文件名。
- 用户修改导图后，标题显示未保存标记；保存成功后，未保存标记消失。
- `npm test` 和 `npm run build` 通过。

## 风险与取舍

- 撤销 / 重做如果把视图平移缩放也纳入历史，容易让用户按撤销时只看到画布移动而不是内容恢复。本轮先只覆盖导图内容与结构。
- 备注输入可能产生过多历史记录。本轮可以按输入变更提交历史，但实现时应避免一次按键生成难以使用的大量撤销步骤。
- 关闭应用确认依赖 Tauri 窗口事件；浏览器开发模式下可以只验证新建和打开确认流程。
- 按钮式确认需要替代 `window.prompt`，但不需要引入复杂弹窗库，优先使用项目内简单组件实现。
