# 轻量右侧导图桌面应用设计

## 背景

用户经常使用 MindManager 绘制右侧导图，但认为 MindManager 太重。第一版目标是实现一个轻量的 Windows 桌面导图工具，专注固定右向树导图的创建、编辑、保存和重新打开。

## 目标

- 提供 Windows 桌面应用，启动快、包体相对轻量。
- 支持固定右向树导图：中心主题在左，所有分支向右展开。
- 支持完整基础编辑闭环：节点编辑、键盘操作、拖拽整理、折叠/展开、备注、本地保存和打开。
- 使用 JSON 文件保存导图数据，便于调试、迁移和后续扩展。

## 非目标

- 不支持自由画布。
- 不支持图片、SVG、Markdown 等导出格式。
- 不支持云同步、多人协作、多标签页。
- 不支持主题样式、图标、附件、富文本备注。
- 不支持 macOS、Linux 打包。

## 技术路线

采用 Tauri + React + TypeScript：

- Tauri 负责 Windows 桌面窗口、本地文件读写、打开/保存对话框和关闭前拦截。
- React 负责 UI、快捷键、节点编辑、备注面板和画布交互。
- TypeScript 统一约束节点、文档和树操作的数据类型。
- 第一版不引入复杂画布引擎；使用 SVG 或 HTML/CSS 组合渲染固定右向树节点与连线。

推荐优先使用自动布局，而不是保存节点坐标。这样可以降低拖拽后的布局复杂度，并让 JSON 文件更稳定。

## 核心界面

应用由四个主要区域组成：

1. 顶部工具栏
   - 新建、打开、保存、另存为。
   - 显示当前文件名和未保存状态。

2. 导图画布
   - 中心主题固定在左侧起点。
   - 子节点全部向右展开。
   - 支持鼠标平移和缩放。
   - 支持选中节点、双击编辑、拖拽调整结构。

3. 节点
   - 显示节点文本。
   - 有子节点时显示折叠/展开按钮。
   - 折叠后隐藏子树，并显示被隐藏的子节点数量提示。
   - 当前选中节点有明确视觉状态。

4. 备注面板
   - 从选中节点打开。
   - 编辑当前节点的纯文本备注。
   - 备注不直接显示在画布节点上，只随文件保存。

## 交互设计

### 节点编辑

- 选中节点后按 `Enter` 新建同级节点。
- 选中节点后按 `Tab` 新建子节点。
- 双击节点或直接输入文本进入编辑。
- 编辑时按 `Enter` 确认，按 `Esc` 取消。
- 按 `Delete` 删除当前节点及其子树。
- 中心主题不可删除。

### 拖拽整理

- 拖动节点到同级节点之间，可调整同级顺序。
- 拖动节点到另一个节点上，可成为该节点的子节点。
- 不允许把节点拖到自身或自身后代下。
- 拖拽完成后自动重新计算右向树布局。

### 折叠与展开

- 有子节点的节点显示折叠/展开按钮。
- 点击按钮切换 `collapsed` 状态。
- 折叠节点隐藏全部后代节点和连线。
- 折叠节点显示隐藏子节点数量提示。

### 备注

- 选中节点后可打开备注面板。
- 备注为纯文本。
- 修改备注会标记文档为未保存。
- 备注随 JSON 文件保存和恢复。

### 画布视图

- 支持鼠标拖动画布平移。
- 支持滚轮或快捷键缩放。
- 保存文件时记录当前平移和缩放位置。
- 打开文件后恢复视图位置。

## 数据结构

JSON 文件包含文档版本、根节点和视图状态。

```json
{
  "version": 1,
  "root": {
    "id": "root",
    "text": "中心主题",
    "note": "",
    "collapsed": false,
    "children": []
  },
  "viewport": {
    "x": 0,
    "y": 0,
    "zoom": 1
  }
}
```

节点结构：

```ts
type MindMapNode = {
  id: string;
  text: string;
  note: string;
  collapsed: boolean;
  children: MindMapNode[];
};
```

文档结构：

```ts
type MindMapDocument = {
  version: 1;
  root: MindMapNode;
  viewport: {
    x: number;
    y: number;
    zoom: number;
  };
};
```

布局坐标不写入 JSON，由当前树结构和折叠状态实时计算。

## 状态管理

前端维护以下状态：

- 当前导图文档。
- 当前选中节点 ID。
- 当前编辑中的节点 ID 和临时文本。
- 当前打开文件路径。
- 未保存状态。
- 当前画布视图位置。
- 当前备注面板打开状态。

树操作集中在纯函数中实现，包括新增、删除、移动、更新文本、更新备注、切换折叠状态。这样便于单元测试，也能避免 UI 组件直接修改嵌套数据。

## 文件操作

### 新建

- 创建默认文档。
- 如果当前文档有未保存更改，先提示保存、放弃或取消。

### 打开

- 通过 Tauri 打开文件对话框选择 JSON 文件。
- 读取后校验 JSON 格式、版本号和根节点结构。
- 校验失败时显示错误提示，并保留当前文档不变。

### 保存

- `Ctrl+S` 保存当前文档。
- 如果当前没有文件路径，执行另存为。
- 保存成功后清除未保存状态。
- 写入失败时显示错误提示，并保留未保存状态。

### 另存为

- 通过 Tauri 保存文件对话框选择路径。
- 将当前文档写入 JSON 文件。
- 保存成功后更新当前文件路径和窗口标题。

### 关闭

- 如果有未保存更改，关闭窗口前提示保存、放弃或取消。
- 没有未保存更改时直接关闭。

## 错误处理

- 无效 JSON：提示文件无法解析。
- 不支持的版本号：提示当前应用不支持该文件版本。
- 数据结构不完整：提示文件不是有效导图文件。
- 文件读取失败：提示无法读取文件。
- 文件写入失败：提示无法保存文件。
- 非法拖拽：忽略操作，并保持原结构不变。

所有错误提示应使用用户可理解的文本，不静默失败。

## 测试计划

### 单元测试

覆盖树操作和序列化逻辑：

- 新增同级节点。
- 新增子节点。
- 删除节点及子树。
- 阻止删除根节点。
- 移动节点到同级不同位置。
- 移动节点到其他父节点下。
- 阻止移动节点到自身或自身后代下。
- 更新节点文本。
- 更新节点备注。
- 切换折叠状态。
- 序列化和反序列化 JSON 文档。

### 交互测试

覆盖主要用户路径：

- 使用 `Enter` 和 `Tab` 创建节点。
- 双击编辑节点文本。
- 删除选中节点。
- 拖拽调整节点顺序。
- 拖拽改变父子关系。
- 折叠和展开节点。
- 编辑备注并保存。
- 打开已保存文件后恢复节点、备注、折叠状态和视图位置。
- 有未保存更改时关闭窗口会提示。

### 手动验收

在 Windows 上完成以下流程：

1. 启动应用。
2. 创建一张包含多级节点的右向导图。
3. 使用键盘新增同级和子级节点。
4. 拖拽调整节点顺序和父子关系。
5. 给至少一个节点添加备注。
6. 折叠一个包含子节点的分支。
7. 保存为 JSON 文件。
8. 关闭并重新打开应用。
9. 打开 JSON 文件。
10. 确认节点结构、文本、备注、折叠状态和视图位置已恢复。

## MVP 验收标准

- 应用可在 Windows 上作为桌面程序运行。
- 用户可以创建、编辑、删除和拖拽整理固定右向树节点。
- 用户可以折叠和展开分支。
- 用户可以为节点编辑纯文本备注。
- 用户可以保存 JSON 文件并重新打开。
- 重新打开后能恢复节点结构、节点文本、备注、折叠状态和视图位置。
- 无效文件、保存失败和未保存关闭都有明确提示。
